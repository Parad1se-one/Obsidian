## 📋 精读框架（8 模块）

### 0) Executive Overview ⭐
5 个核心问题，1 句话回答：
- 研究问题是什么？为何重要？
- 核心想法/装置是什么？
- 与基线相比的主要改进点？
- 主效果与适用边界（含代表性数字）？
- 总体判断：值得/不值得深入阅读（12 字理由）

### 1) Background（相关脉络与定位）
- 问题定义、评估范式、常见基线、历史演进、现状空白
- 一句话词汇表（跨域术语解释）

### 2) Motivation（痛点与研究空白）
- 作者声称的具体缺口
- 既有方法的不足："主张 → 证据 → 评述"
- 受控变量与评测公平性检查

### 3) Claimed Contributions
- C1..Ck：每条贡献的原创性、可复用价值评估
- 证据锚点 + 可复用产出（代码/数据/理论/工具）

### 4) Method（可落地复述）
- 核心思想、关键假设、形式化与目标
- 算法/流程（N 步要点 + 伪代码 + 复杂度）
- 设计权衡、失败模式

### 5) Results（证据、效应大小与鲁棒性）
- 数据与设置、基线与公平性
- 主要结果（表格：任务/数据/指标/增益/方差/页码）
- 消融与归因、鲁棒与外推、可复现性

### 6) Limitations & Threats to Validity
- 内在局限（数据偏置/假设强度/可扩展性/统计功效）
- 有效性威胁（内部/外部/构造/结论）
- 伦理与风险（隐私/滥用/安全/社会影响）

### 7) Future Directions
≥5 条未来改进方向

### 8) For Busy Readers
- 3 条 Takeaway（每条≤20 字）
- 复现建议：是/否 + 理由 + 优先级 + 预计工作量

## ✅ 质量检查清单

### 自我校核
- [ ] 每条主张都有来源标注 [sec.X] 或 [p.X]
- [ ] 关键数值/设置可定位到页/节
- [ ] 结论以效应大小与不确定性表述
- [ ] 适用边界与失败条件有明确交代
- [ ] 表格未含长句（长句写在正文）
- [ ] 信息缺失处标注"证据不足"，不臆测

### 风格检查
- [ ] 学术而简洁
- [ ] 先结论后理由
- [ ] 多用项目符号与短句
- [ ] 中文为主，必要术语保留英文
- [ ] 跨域术语提供"一句话词汇表"
