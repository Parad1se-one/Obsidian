# LimiX（极数）数据大模型详细介绍

## 🎯 核心突破

### 为什么需要 LimiX？

当前大语言模型（LLM）在非结构化数据（文本、图像）上表现卓越，但在**结构化表格数据**上频频失手：

- 电网调度、用户建模、通信日志等关键系统核心数据都以表格形式存在
- 传统深度学习方法（TabNet、SAINT、FT-Transformer）难以超越 XGBoost、CatBoost 等树模型
- LimiX 填补了这一技术空白，实现**从 LLM"并跑"到 LDM（数据大模型）"领跑"**的跨越

**关键优势**：

- ✅ **多任务支持**：分类、回归、缺失值插补一模型通吃

## 🏗️ 技术架构

### 模型设计

LimiX 基于 **Transformer 架构**，针对结构化数据建模进行了深度优化：

```
┌─────────────────────────────────────────────────────┐
│                    LimiX 架构                        │
├─────────────────────────────────────────────────────┤
│  输入层                                              │
│  ├── 特征 Embedding                                  │
│  └── 目标 Embedding                                  │
├─────────────────────────────────────────────────────┤
│  主模块（双重注意力机制）                             │
│  ├── 样本维度注意力 → 聚焦关键样本                   │
│  └── 特征维度注意力 → 聚焦关键特征                   │
├─────────────────────────────────────────────────────┤
│  输出头                                              │
│  ├── 回归头（Regression Head）                       │
│  └── 分类头（Classification Head）                   │
└─────────────────────────────────────────────────────┘
```

### 核心创新

#### 1️⃣ RBF 嵌入层（RaBEL）

**问题诊断**：传统模型（TabPFN-v2、LimiX-16M、FT-Transformer）采用"线性映射 + 列 ID"方案，存在**低秩塌陷**问题：
- 模型浅层激活高度相关，有效特征种类有限
- 网络早期处于"线性近似"状态，难以识别复杂局部变化
- 梯度利用率低，浪费表达能力

**解决方案**：引入 **RaBEL（Radial Basis Embedding Layer）**
- 在嵌入阶段就引入非线性表达能力
- 在数轴上布置多个"感受野"，不同取值区间形成独立激活
- 浅层即可识别非平滑关系，从根本上解决低秩问题

#### 2️⃣ 因果稳定学习理论

基于清华大学崔鹏教授团队构建的**全球首个"因果稳定学习理论"体系**：
- 融合**因果推理**技术，理解数据内在因果机制
- 构建"通用世界模型"，实现"一个模型赋能千行百业"
- 具备**可解释性**，打开模型"黑盒"

#### 3️⃣ 合成数据生成训练

LimiX 预训练完全依赖**生成数据**而非真实世界数据：
- 采用**结构因果图（SCG）** 数据生成方式
- 初始样本在有向无环图上传播，模拟真实因果依赖关系
- 保证数据因果多样性，提升可控性与泛化能力

#### 4️⃣ 掩码重构机制

- 对数据联合分布建模，提高通用性
- 训练过程中随机掩码操作
- 根据特征间因果依赖关系重构缺失数据
- 增强对特征交互模式的建模能力
