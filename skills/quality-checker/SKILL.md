# Quality Checker Skill - 输出质量自检

## Overview
自动评估生成内容的质量，不及格则重新生成。

**人设：** 小虾 🦐 - "垃圾输出就是害人，我宁可重做"

**版本：** v1.0 (2026-03-05)

**适用场景：** 投资决策、学术分析、重要报告等高风险输出

---

## Core Philosophy

### 质量红线
| 类型 | 红线 | 后果 |
|------|------|------|
| **数据造假** | 编造数据、来源不明 | 直接不及格 |
| **信息缺失** | 关键字段为空 | 扣分 |
| **建议空洞** | 无逻辑支撑的泛泛而谈 | 扣分 |
| **格式混乱** | 结构不完整 | 扣分 |

### 评估原则
1. **真实性优先** - 宁可说"不知道"，也不编造
2. **来源透明** - 每条数据都要有来源
3. **逻辑可追溯** - 建议必须有推理链条
4. **用户第一** - 垃圾输出就是浪费用户时间

---

## Evaluation Framework

### 通用评估维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| **数据真实性** | 40% | 100% 真实且有来源=100 分，有编造=0 分 |
| **信息完整性** | 25% | 关键字段覆盖率 |
| **建议可操作性** | 20% | 是否有具体行动指引 |
| **格式规范性** | 15% | 结构、排版、可读性 |

### 评分细则

#### 数据真实性 (40%)
| 情况 | 得分 |
|------|------|
| 所有数据有明确来源，可验证 | 100 |
| 大部分数据有来源，少量缺失 | 70 |
| 数据来源模糊 | 40 |
| 有编造数据 | 0 (直接不及格) |

#### 信息完整性 (25%)
| 覆盖率 | 得分 |
|--------|------|
| ≥90% | 100 |
| 80-89% | 85 |
| 70-79% | 70 |
| 60-69% | 55 |
| <60% | 40 |

#### 建议可操作性 (20%)
| 质量 | 得分 |
|------|------|
| 具体、可执行、有逻辑支撑 | 90-100 |
| 有方向但不够具体 | 70-89 |
| 泛泛而谈 | 50-69 |
| 空洞无物 | 0-49 |

#### 格式规范性 (15%)
| 质量 | 得分 |
|------|------|
| 结构清晰、排版美观 | 90-100 |
| 结构完整、排版一般 | 75-89 |
| 结构有缺失 | 50-74 |
| 混乱 | 0-49 |

---

## Quality Thresholds

| 等级 | 分数 | 处理方式 |
|------|------|----------|
| 🟢 优秀 | ≥90 | 直接输出 |
| 🟡 良好 | 80-89 | 输出 + 标注改进点 |
| 🟠 及格 | 70-79 | 询问用户是否接受 |
| 🔴 不及格 | <70 | **必须重做** |

## Usage

### 命令行
```bash
./quality-checker.sh <内容文件> <类型>
# 类型：financial|academic|report|general
```

### 输出格式
```markdown
## 质量评估报告

| 维度 | 得分 | 说明 |

**综合得分：83/100** 🟡 良好

**改进建议：**
1. 补充 XX 数据
2. 细化 XX 建议

**处理决定：** 通过（标注改进点）
```

## Integration

### 与工作流集成

```bash
# 1. 生成内容
./financial-daily.sh 2026-03-05

# 2. 质量检查
./quality-checker.sh daily/financial-news/2026-03-05.md financial

# 3. 根据结果决定
# - ≥80 分：推送给用户
# - <80 分：重新生成
```

### 自动重试机制

```bash
MAX_RETRIES=3
RETRY_COUNT=0

while [ $RETRY_COUNT -lt $MAX_RETRIES ]; do
    # 生成内容
    generate_content
    
    # 质量检查
    SCORE=$(./quality-checker.sh output.md)
    
    if [ $SCORE -ge 80 ]; then
        echo "✅ 质量检查通过 ($SCORE 分)"
        break
    else
        echo "❌ 质量检查未通过 ($SCORE 分)，重做中..."
        RETRY_COUNT=$((RETRY_COUNT + 1))
    fi
done

if [ $RETRY_COUNT -eq $MAX_RETRIES ]; then
    echo "⚠️  已达到最大重试次数，输出当前版本（附质量报告）"
fi
```

## Templates

```markdown
## 财经日报质量检查清单

### 数据真实性
- [ ] 行情数据来自腾讯财经/新浪财经 API
- [ ] 新闻来自真实网站抓取（东方财富、新浪财经等）

- [ ] 所有数据标注来源

- [ ] 4 大指数行情（上证、深证、恒生、纳斯达克）

- [ ] ≥3 条公司动态
- [ ] 行业热点（领涨/领跌板块）
- [ ] 投资建议（具体方向 + 理由 + 风险）
- [ ] 明日关注事项

### 建议可操作性
- [ ] 投资建议有逻辑支撑
- [ ] 风险提示明确
- [ ] 仓位建议具体
- [ ] 止损建议明确

### 格式规范性
- [ ] 结构完整（8 个模块）
- [ ] 表格清晰
- [ ] 免责声明醒目
- [ ] 数据来源标注
```

## 论文精读质量检查清单

### 数据真实性
