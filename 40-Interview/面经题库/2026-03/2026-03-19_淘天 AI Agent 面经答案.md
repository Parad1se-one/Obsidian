### 核心组件说明

| 组件 | 职责 | 技术选型 |
|------|------|---------|
| **API Gateway** | 认证、限流、路由 | Kong / APISIX |
| **Planning Agent** | 任务规划与分解 | LLM + Prompt Engineering |
| **Router Agent** | 意图识别与路由 | 语义路由 + 分类模型 |
| **Worker Agent** | 具体任务执行 | 子 Agent 集群 |
| **Review Agent** | 质量审核与校验 | 规则引擎 + LLM |
| **LLM Engine** | 模型推理服务 | vLLM / TGI |
| **RAG Engine** | 检索增强生成 | Milvus + BM25 |
| **Memory Manager** | 上下文与记忆管理 | Redis + Vector DB |
| **Tool Registry** | 工具注册与发现 | MCP Protocol |

### 数据流转

```
1. 用户请求 → API Gateway (认证/限流)
2. → Planning Agent (任务分解)
3. → Router Agent (路由到对应 Worker)
4. → Worker Agent (执行具体任务)
   ├─ 调用 LLM Engine (推理)
   ├─ 查询 RAG Engine (检索知识)
   ├─ 读取 Memory Manager (上下文)
   └─ 调用 Tool Registry (外部工具)
5. → Review Agent (质量审核)
6. → 返回用户
```

## Q2: Agent 的"规划 - 执行 - 反思"闭环如何实现？

### 实现架构

```python
class PlanningExecutionReflection:
    def __init__(self):
        self.planner = LLMAgent("planner")
        self.executor = ToolExecutor()
        self.reflector = LLMAgent("reflector")
        self.memory = MemoryManager()
        
    async def execute(self, task: str) -> dict:
        # === 阶段 1: 规划 ===
        plan = await self.planner.generate(f"""
        任务：{task}
        
        请制定详细执行计划：
        1. 任务目标是什么？
        2. 需要分解为哪些子任务？
        3. 子任务的依赖关系？
        4. 每个子任务需要什么工具？
        5. 预期输出是什么？
        """)
        
        # === 阶段 2: 执行 ===
        results = []
        for step in plan.steps:
            result = await self.executor.execute(step)
            results.append({
                "step": step,
                "result": result,
                "success": result.success
            })
            
            # 执行失败时动态调整
            if not result.success:
                plan = await self._replan(plan, step, result.error)
        
        # === 阶段 3: 反思 ===
        reflection = await self.reflector.generate(f"""
        原始任务：{task}
        执行计划：{plan}
        执行结果：{results}
        
        请反思：
        1. 哪些步骤执行得好？
        2. 哪些步骤出现问题？为什么？
        3. 如果重新执行，会如何改进？
        4. 从这次执行中学到了什么？
        """)
        
        # === 阶段 4: 记忆存储 ===
        await self.memory.store_experience({
            "task": task,
            "plan": plan,
            "results": results,
            "reflection": reflection
        })
        
        return {
            "plan": plan,
            "results": results,
            "reflection": reflection
        }
    
    async def _replan(self, plan, failed_step, error):
        """动态重规划"""
        new_plan = await self.planner.generate(f"""
        原计划：{plan}
        失败步骤：{failed_step}
        错误信息：{error}
        
        请调整计划以绕过或解决这个问题。
        """)
        return new_plan
```

### 关键设计点

| 环节 | 设计要点 | 实现方式 |
|------|---------|---------|
| **规划** | 任务分解、依赖分析 | LLM + CoT Prompt |
| **执行** | 工具调用、错误处理 | Tool Executor + 重试机制 |
| **反思** | 经验总结、持续改进 | LLM + 结构化反思模板 |
| **记忆** | 经验存储、检索复用 | Vector DB + 元数据索引 |

## Q3: 多轮对话中，记忆如何分级管理？缓存策略怎么设计？

### 记忆分级架构

```

│ L1: Working Memory (工作记忆) - 毫秒级访问                       │
├─────────────────────────────────────────────────────────────────┤
│ 内容：最近 5-10 轮对话                                            │
│ 存储：内存 (Redis)                                              │

│ 容量：~10KB                                                     │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────┤
│ 内容：历史对话片段 (向量化)                                      │
│ 存储：向量数据库 (Milvus/Chroma)                                │
│ 过期：按业务需求 (通常 90 天)                                     │
│ 容量：~10MB/用户                                                │
└─────────────────────────────────────────────────────────────────┘
```

### 缓存策略设计

```python
class MemoryCacheManager:
    def __init__(self):
        # L1 缓存：Redis (热数据)
        self.l1_cache = RedisClient(
            host="redis-cluster",
            ttl=1800,  # 30 分钟
            max_memory="10GB"
        )
        
        # L2 缓存：本地内存 (极热数据)
        self.l2_cache = LRUCache(
            max_size=1000,
            ttl=300  # 5 分钟
        )
        
        # 向量检索缓存
        self.vector_cache = VectorCache(
            backend="milvus",
            cache_size=10000
        )
    
    async def get_conversation_context(self, session_id: str) -> list:
        """获取对话上下文"""
        # 1. 检查 L2 缓存 (最快)
        if session_id in self.l2_cache:
            return self.l2_cache[session_id]
        
        # 2. 检查 L1 缓存 (Redis)
        key = f"conversation:{session_id}"
        cached = await self.l1_cache.get(key)
        if cached:
            # 回填 L2
            self.l2_cache[session_id] = cached
            return cached
        
        # 3. 从数据库加载
        messages = await self._load_from_db(session_id, limit=50)
        
        # 4. 写入缓存
        await self.l1_cache.set(key, messages)
        self.l2_cache[session_id] = messages
        
        return messages
    
    async def store_memory(self, user_id: str, content: str, metadata: dict):
        """存储记忆"""
        # 1. 向量化
        embedding = await self.embed(content)
        
        # 2. 存储到向量数据库
        await self.vector_db.insert({
            "user_id": user_id,
            "content": content,
            "embedding": embedding,
            "metadata": metadata,
            "timestamp": time.time()
        })
        
        # 3. 更新语义记忆 (关键信息提取)
        entities = await self._extract_entities(content)
        await self._update_semantic_memory(user_id, entities)
```

### 缓存淘汰策略

| 层级 | 策略 | 触发条件 |
|------|------|---------|
| **L1 (Redis)** | TTL + LRU | 30 分钟过期 或 内存不足 |
| **L2 (内存)** | LRU | 容量满时淘汰最久未使用 |
| **向量缓存** | LFU | 淘汰最低频查询 |

## Q5: Function Calling 的可靠性如何保障？如果模型一直调用同一个错误工具怎么办？

### 可靠性保障体系

```python
class FunctionCallingReliability:
    def __init__(self):
        self.tool_registry = ToolRegistry()
        self.validator = ToolValidator()
        self.circuit_breaker = CircuitBreaker()
        self.retry_policy = RetryPolicy()
        
    async def execute_with_reliability(self, tool_name: str, args: dict) -> any:
        # 1. 工具存在性校验
        tool = self.tool_registry.get(tool_name)
        if not tool:
            raise ToolNotFoundError(f"工具 {tool_name} 不存在")
        
        # 2. 参数校验
        validation = await self.validator.validate(tool_name, args)
        if not validation.valid:
            raise ValidationError(f"参数校验失败：{validation.errors}")
        
        # 3. 熔断器检查
        if self.circuit_breaker.is_open(tool_name):
            raise CircuitBreakerError(f"工具 {tool_name} 熔断中")
        
        # 4. 执行 (带重试)
        try:
            result = await self.retry_policy.execute(
                func=tool.execute,
                args=args,
                max_retries=3,
                backoff="exponential"
            )
            return result
        except Exception as e:
            # 5. 失败处理
            self.circuit_breaker.record_failure(tool_name)
            
            # 6. 降级策略
            fallback = self.tool_registry.get_fallback(tool_name)
            if fallback:
                return await fallback.execute(args)
            raise
    
    async def detect_loop(self, call_history: list) -> bool:
        """检测重复调用循环"""
        if len(call_history) < 3:
            return False
        
        # 检查最近 N 次调用是否重复
        recent_calls = call_history[-5:]
        tool_counts = Counter([c["tool_name"] for c in recent_calls])
        
        # 同一工具调用超过阈值
        for tool_name, count in tool_counts.items():
            if count >= 4:
                return True
        return False
```

### 防止重复错误调用

**策略 1：调用频率限制**

```python
class CallRateLimiter:
    def __init__(self):
        self.call_counts = defaultdict(list)  # tool_name → [timestamps]
        self.limits = {
            "default": {"max_calls": 10, "window_seconds": 60},
            "expensive": {"max_calls": 3, "window_seconds": 60}
        }
    
    def check_rate_limit(self, tool_name: str) -> bool:
        now = time.time()
        window = 60  # 1 分钟窗口
        
        # 清理过期记录
        self.call_counts[tool_name] = [
            t for t in self.call_counts[tool_name]
            if now - t < window
        ]
        
        # 检查是否超限
        limit = self.limits.get(tool_name, self.limits["default"])
        if len(self.call_counts[tool_name]) >= limit["max_calls"]:
            return False
        
        # 记录调用
        self.call_counts[tool_name].append(now)
        return True
```

**策略 2：智能重试与降级**

```python
class SmartRetry:
    async def execute_with_smart_retry(self, tool_name: str, args: dict):
        last_error = None
        
        for attempt in range(3):
            try:
                return await self.tool_registry.execute(tool_name, args)
            except TimeoutError as e:
                last_error = e
                # 超时：指数退避
                await asyncio.sleep(2 ** attempt)
            except ValidationError as e:
                # 参数错误：不重试，修正参数
                args = await self._fix_args(tool_name, args, e)
            except RateLimitError as e:
                # 限流：等待后重试
                await asyncio.sleep(e.retry_after)
        
        # 所有重试失败：降级
        return await self._fallback(tool_name, args, last_error)
```

**策略 3：调用模式监控**

```python
class CallPatternMonitor:
    def __init__(self):
        self.call_history = deque(maxlen=1000)
        
    def record_call(self, session_id: str, tool_name: str, result: str):
        self.call_history.append({
            "session_id": session_id,
            "tool_name": tool_name,
            "result": result,
            "timestamp": time.time()
        })
    
    def detect_anomaly(self, session_id: str) -> dict:
        """检测异常调用模式"""
        recent = [c for c in self.call_history if c["session_id"] == session_id][-20:]
        
        # 检测重复失败
        failures = [c for c in recent if c["result"] == "error"]
        failure_rate = len(failures) / len(recent) if recent else 0
        
        # 检测单一工具重复调用
        tool_counts = Counter([c["tool_name"] for c in recent])
        max_tool_ratio = max(tool_counts.values()) / len(recent) if recent else 0
        
        return {
            "is_anomaly": failure_rate > 0.5 or max_tool_ratio > 0.8,
            "failure_rate": failure_rate,
            "max_tool_ratio": max_tool_ratio
        }
```

## Q6: 工具的描述 (description) 怎么写才能让模型准确调用？给出淘天场景示例

### Description 编写原则

**STAR 法则：**

| 要素 | 说明 | 示例 |
|------|------|------|
| **S**ituation | 使用场景 | "当用户查询商品时..." |
| **T**ask | 任务目标 | "...获取商品详细信息..." |
| **A**ction | 执行动作 | "...调用商品查询 API..." |
| **R**esult | 预期结果 | "...返回商品名称、价格、库存" |

## Q7: RAG 系统中，chunk 大小怎么确定？不同业务场景的策略差异？

### Chunk 大小选择原则

| 因素 | 影响 | 建议 |
|------|------|------|
| **模型上下文窗口** | 窗口越大，chunk 可越大 | GPT-4: 4K, Claude: 100K |
| **检索精度** | chunk 越小，定位越精确 | 但可能丢失上下文 |
| **语义完整性** | chunk 需保持语义完整 | 避免在句子中间切分 |
| **业务场景** | 不同场景需求不同 | 见下表 |

### 不同业务场景策略

| 场景 | Chunk 大小 | 重叠 | 理由 |
|------|----------|------|------|
| **商品详情** | 512-1024 tokens | 50-100 | 商品属性独立，小 chunk 精确 |
| **客服 FAQ** | 256-512 tokens | 50 | 问答对独立，小 chunk 足够 |
| **技术文档** | 1024-2048 tokens | 100-200 | 需要上下文理解 |
| **法律合同** | 512-1024 tokens | 100 | 条款独立但需精确引用 |
| **新闻文章** | 1024-2048 tokens | 200 | 保持叙事完整性 |
| **代码文档** | 512-1024 tokens | 50 | 函数/类为单位 |

### 智能分块策略

```python
class SmartChunker:
    def __init__(self, strategy: str = "auto"):
        self.strategy = strategy
        self.chunk_sizes = {
            "product": {"size": 512, "overlap": 50},
            "faq": {"size": 256, "overlap": 50},
            "doc": {"size": 1024, "overlap": 100},
            "code": {"size": 512, "overlap": 50}
        }
    
    def chunk(self, text: str, doc_type: str = "auto") -> list:
        # 1. 自动识别文档类型
        if doc_type == "auto":
            doc_type = self._detect_type(text)
        
        # 2. 获取分块参数
        params = self.chunk_sizes.get(doc_type, self.chunk_sizes["doc"])
        
        # 3. 语义分块 (按段落/句子)
        chunks = self._semantic_split(text, params["size"], params["overlap"])
        
        # 4. 质量检查
        chunks = [c for c in chunks if len(c) > 50]  # 过滤太短 chunk
        
        return chunks
    
    def _semantic_split(self, text: str, max_size: int, overlap: int) -> list:
        """语义分块"""
        # 1. 按段落分割
        paragraphs = text.split("\n\n")
        
        chunks = []
        current_chunk = ""
        
        for para in paragraphs:
            if len(current_chunk) + len(para) < max_size:
                current_chunk += "\n\n" + para
            else:
                # 当前 chunk 已满，保存并开始新 chunk
                if current_chunk:
                    chunks.append(current_chunk)
                current_chunk = para
        
        if current_chunk:
            chunks.append(current_chunk)
        
        # 添加重叠
        if overlap > 0 and len(chunks) > 1:
            chunks = self._add_overlap(chunks, overlap)
        
        return chunks
```

```python
async def execute_parallel(self, steps: list):
    # 1. 分析依赖关系
    dag = self._build_dependency_graph(steps)
    
    # 2. 拓扑排序，找出可并行的步骤
    levels = self._topological_sort(dag)
    
    # 3. 按层级并行执行
    results = {}
    for level in levels:
        # 同一层级内并行
        tasks = [
            self.execute_tool(step).then(lambda r, s=step: results.update({s: r}))
            for step in level
        ]
        await asyncio.gather(*tasks)
    
    return results
```

## Q10: 高并发场景下，如何设计 Agent 服务的弹性伸缩策略？

### 弹性伸缩架构

```python
class AutoScaler:
    def __init__(self):
        self.metrics_client = PrometheusClient()
        self.k8s_client = K8sClient()
        
        # 伸缩策略
        self.scaling_policy = {
            "min_replicas": 2,
            "max_replicas": 100,
            "target_cpu": 70,
            "target_memory": 80,
            "target_rps_per_pod": 100,
            "scale_up_cooldown": 60,  # 秒
            "scale_down_cooldown": 300  # 秒
        }
    
    async def scale(self):
        # 1. 采集指标
        metrics = await self._collect_metrics()
        
        # 2. 计算目标副本数
        target_replicas = self._calculate_target_replicas(metrics)
        
        # 3. 执行伸缩
        await self._apply_scaling(target_replicas)
    
    def _calculate_target_replicas(self, metrics: dict) -> int:
        current = metrics["current_replicas"]
        
        # CPU 指标
        cpu_ratio = metrics["avg_cpu"] / self.scaling_policy["target_cpu"]
        cpu_target = int(current * cpu_ratio)
        
        # RPS 指标
        rps_target = int(metrics["total_rps"] / self.scaling_policy["target_rps_per_pod"])
        
        # 取最大值
        target = max(cpu_target, rps_target)
        
        # 限制范围
        target = max(self.scaling_policy["min_replicas"], 
                    min(target, self.scaling_policy["max_replicas"]))
        
        return target
```

```python
class DistributedTracing:
    def __init__(self):
        self.tracer = OpenTelemetryTracer(
            service_name="agent-system",
            exporter="jaeger"
        )
    
    async def process_request(self, request: dict):
        # 1. 创建 Trace
        trace_id = str(uuid4())
        span_id = str(uuid4())
        
        # 2. 注入上下文
        context = {
            "trace_id": trace_id,
            "span_id": span_id,
            "parent_span_id": None
        }
        
        # 3. 创建根 Span
        with self.tracer.start_span("agent_request", context=context) as root_span:
            root_span.set_attribute("request_id", request["id"])
            root_span.set_attribute("user_id", request["user_id"])
            
            # 4. 执行 Planning
            with self.tracer.start_span("planning", parent=root_span) as planning_span:
                plan = await self.planner.execute(request)
                planning_span.set_attribute("plan_steps", len(plan.steps))
            
            # 5. 执行各个步骤
            results = []
            for i, step in enumerate(plan.steps):
                with self.tracer.start_span(f"step_{i}", parent=root_span) as step_span:
                    step_span.set_attribute("step_type", step.type)
                    step_span.set_attribute("tool_name", step.tool)
                    
                    result = await self.execute_step(step)
                    results.append(result)
                    
                    step_span.set_attribute("success", result.success)
            
            # 6. 完成 Trace
            root_span.set_attribute("total_steps", len(results))
            root_span.set_attribute("success", all(r.success for r in results))
        
        return results
```
