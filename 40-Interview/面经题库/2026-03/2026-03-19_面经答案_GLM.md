### 1. 语义路由怎么实现？怎么评估效果？

**实现方案：**

```python
# 语义路由核心架构
class SemanticRouter:
    def __init__(self):
        self.embedding_model = "text-embedding-3-small"
        self.routes = {}  # 路由表：意图→处理函数
        
    def register_route(self, intent: str, handler: callable, examples: list):
        """注册路由"""
        self.routes[intent] = {
            'handler': handler,
            'embeddings': self._embed_batch(examples),
            'examples': examples
        }
    
    def route(self, query: str) -> str:
        """语义路由决策"""
        query_emb = self._embed(query)
        
        # 计算与各意图的相似度
        scores = {}
        for intent, data in self.routes.items():
            scores[intent] = cosine_similarity(query_emb, data['embeddings'].mean(axis=0))
        
        # 返回最高分意图（超过阈值）
        best_intent = max(scores, key=scores.get)
        if scores[best_intent] > 0.75:
            return best_intent
        return "default"  # 默认路由
```

**评估指标：**

| 指标 | 计算方法 | 目标值 |
|------|---------|--------|
| 准确率 | 正确路由数/总请求数 | >95% |
| 延迟 | P99 路由决策时间 | <50ms |
| 覆盖率 | 有明确意图的请求占比 | >90% |
| 人工复核率 | 需要人工干预的比例 | <5% |

**评估方法：**
1. **离线测试集**：构建 500+ 标注样本，计算准确率/召回率
2. **A/B 测试**：新旧路由策略对比

│  │Agent │  │Agent │  │Agent │                          │
│  └──────┘  └──────┘  └──────┘                          │
│  - 有独立状态和记忆                                       │

└─────────────────────────────────────────────────────────┘

**选择子 Agent 的原因：**

| 场景 | Skill | 子 Agent | 选择理由 |
|------|-------|----------|---------|
| 简单工具调用 | ✅ | ❌ | Skill 更轻量 |
| 复杂任务分解 | ❌ | ✅ | 子 Agent 可独立规划 |
| 需要记忆/状态 | ❌ | ✅ | 子 Agent 有独立上下文 |
| 多轮对话 | ❌ | ✅ | 子 Agent 可维护对话状态 |

**我们的选择**：采用**混合架构**
- 简单操作 → Skill（函数调用）
- 复杂任务 → 子 Agent（独立决策）

### 3. 上下文管理？如何避免遗忘或幻觉？

**上下文管理架构：**

```python
class ContextManager:
    def __init__(self):
        self.short_term = []  # 短期记忆（最近 N 轮）
        self.long_term = VectorStore()  # 长期记忆（向量检索）
        self.summary = ""  # 对话摘要
        self.entities = {}  # 关键实体
        
    def add_message(self, role: str, content: str):
        """添加消息"""
        # 1. 加入短期记忆
        self.short_term.append({"role": role, "content": content})
        
        # 2. 提取关键信息到长期记忆
        if len(content) > 100:
            self._extract_and_store(content)
        
        # 3. 定期摘要（每 10 轮）
        if len(self.short_term) % 10 == 0:
            self.summary = self._generate_summary()
    
    def get_context(self, max_tokens: int) -> list:
        """构建上下文"""
        context = []
        
        # 1. 系统提示 + 摘要
        context.append({"role": "system", "content": self.summary})
        
        # 2. 最近 N 轮对话
        context.extend(self.short_term[-5:])
        
        # 3. 相关长期记忆
        if self.long_term:
            relevant = self.long_term.search(query=context[-1]["content"], k=3)
            context.append({"role": "system", "content": f"相关记忆：{relevant}"})
        
        return truncate_to_tokens(context, max_tokens)
```

**避免遗忘：**
1. **分层记忆**：短期（完整）+ 长期（向量检索）+ 摘要（压缩）
2. **关键实体追踪**：人名、地名、时间等单独存储
3. **定期回顾**：每 N 轮对话后生成摘要

**避免幻觉：**
1. **引用溯源**：所有回答必须标注信息来源
2. **置信度阈值**：低于阈值时明确说"不确定"
3. **事实核查**：关键信息通过工具验证（搜索/数据库）
4. **边界声明**：明确说明知识截止日期

class AgentCommunication:
    def __init__(self):
        self.message_bus = MessageBus()  # 消息总线
        self.blackboard = Blackboard()   # 共享黑板
        self.registry = AgentRegistry()  # Agent 注册表
        
    def broadcast(self, sender: str, message: dict):
        """广播消息"""
        self.message_bus.publish("global", {
            "from": sender,
            "content": message,
            "timestamp": time.time()
        })
    
    def direct_message(self, from_agent: str, to_agent: str, message: dict):
        """点对点消息"""
        self.message_bus.publish(f"agent:{to_agent}", {
            "from": from_agent,
            "content": message
        })
    
    def write_blackboard(self, key: str, value: any, ttl: int = 3600):
        """写入共享黑板"""
        self.blackboard.set(key, value, ttl=ttl)
    
    def read_blackboard(self, key: str) -> any:
        """读取共享黑板"""
        return self.blackboard.get(key)
```

class ContentSafetyFilter:
    def __init__(self):
        self.sensitive_words = self._load_sensitive_words()
        self.classifier = load_model("safety-classifier")
        self.regex_patterns = self._load_regex_patterns()
        
    def filter_input(self, text: str) -> tuple[bool, str]:
        """输入过滤"""
        # 1. 敏感词匹配
        for word in self.sensitive_words:
            if word in text:
                return False, f"包含敏感词：{word}"
        
        # 2. 正则匹配（电话、身份证等）
        for pattern in self.regex_patterns:
            if re.search(pattern, text):
                return False, "包含敏感信息模式"
        
        # 3. 提示词注入检测
        if self._detect_injection(text):
            return False, "检测到提示词注入"
        
        return True, ""
    
    def filter_output(self, text: str) -> tuple[bool, str]:
        """输出过滤"""
        # 1. 敏感词过滤
        for word in self.sensitive_words:
            text = text.replace(word, "***")
        
        # 2. 安全分类
        risk_score = self.classifier.predict(text)
        if risk_score > 0.8:
            return False, "内容风险过高"
        
        # 3. 涉密检测
        if self._detect_secret(text):
            return False, "可能包含涉密内容"
        
        return True, text
```

## 三、手撕题：Agent 调用环检测

### 题目：Agent 可以调用另一个 Agent，如何检测环？加深度限制？

**解题思路：类似力扣课程表（拓扑排序/DFS）**

```python
class AgentCallGraph:
    def __init__(self):
        self.graph = defaultdict(list)  # 邻接表
        self.max_depth = 5  # 最大调用深度
        
    def add_call(self, caller: str, callee: str):
        """添加调用关系"""
        self.graph[caller].append(callee)
    
    def has_cycle(self) -> bool:
        """检测是否有环（DFS）"""
        WHITE, GRAY, BLACK = 0, 1, 2
        color = {node: WHITE for node in self.graph}
        
        def dfs(node):
            color[node] = GRAY  # 标记为访问中
            for neighbor in self.graph[node]:
                if color[neighbor] == GRAY:  # 遇到灰色节点→有环
                    return True
                if color[neighbor] == WHITE and dfs(neighbor):
                    return True
            color[node] = BLACK  # 标记为已完成
            return False
        
        for node in self.graph:
            if color[node] == WHITE:
                if dfs(node):
                    return True
        return False
    
    def execute_with_depth_limit(self, agent: str, depth: int = 0) -> any:
        """带深度限制的调用"""
        if depth > self.max_depth:
            raise RecursionError(f"超过最大调用深度 {self.max_depth}")
        
        # 执行 Agent 逻辑
        result = self._execute_agent(agent)
        
        # 处理子调用
        for callee in self.graph[agent]:
            child_result = self.execute_with_depth_limit(callee, depth + 1)
            result = self._merge_results(result, child_result)
        
        return result
```

**测试用例：**

```python

graph1.add_call("A", "B")
graph1.add_call("B", "C")
assert graph1.has_cycle() == False

# 测试 2: 有环
graph2 = AgentCallGraph()
graph2.add_call("A", "B")
graph2.add_call("B", "C")
graph2.add_call("C", "A")  # 环
assert graph2.has_cycle() == True

# 测试 3: 深度限制

graph3.add_call("A", "B")
graph3.add_call("B", "C")
graph3.add_call("C", "D")
graph3.add_call("D", "E")
try:
    graph3.execute_with_depth_limit("A")

```

### 3. Planning & Solve 架构？还了解什么架构？

**Planning & Solve 架构：**

```python
class PlanningAndSolve:
    def __init__(self):
        self.planner = LLMAgent("planner")
        self.solver = LLMAgent("solver")
        
    def execute(self, problem: str) -> str:
        # Step 1: 规划
        plan = self.planner.generate(f"""
        问题：{problem}
        
        请制定解决计划：
        1. 分解为子任务
        2. 确定执行顺序
        3. 识别依赖关系
        """)
        
        # Step 2: 执行
        results = []
        for step in plan.steps:
            result = self.solver.execute(step)
            results.append(result)
        
        # Step 3: 整合
        solution = self.planner.synthesize(plan, results)
        return solution
```

| **ReAct** | Reason + Act 交替 | 可解释性强 | 延迟高 | 复杂推理 |
| **Reflexion** | 自我反思改进 | 持续优化 | 需要多轮 | 迭代任务 |
| **Chain of Thought** | 逐步推理 | 简单直接 | 容易偏离 | 数学/逻辑 |
| **Tree of Thoughts** | 多路径探索 | 全局最优 | 计算量大 | 决策问题 |
| **Planning & Solve** | 先规划后执行 | 结构化 | 灵活性低 | 流程化任务 |

**ReAct 示例：**

```python
# ReAct: Reason → Act → Observe → Repeat
class ReActAgent:
    def execute(self, task: str) -> str:
        thoughts = []
        
        for i in range(max_iterations):
            # Reason
            thought = self.llm.generate(f"""
            任务：{task}
            历史：{thoughts}
            
            思考：当前应该做什么？
            """)
            thoughts.append(thought)
            
            # Act
            action = self.parse_action(thought)
            observation = self.execute_action(action)
            thoughts.append(f"行动：{action}\n观察：{observation}")
            
            # 检查是否完成
            if self.is_complete(thought):
                return self.extract_answer(thought)
        
        return "未能完成任务"
```

**Reflexion 示例：**

```python
# Reflexion: 执行 → 评估 → 反思 → 改进
class ReflexionAgent:
    def execute(self, task: str) -> str:
        best_solution = None
        best_score = 0
        
        for trial in range(max_trials):
            # 执行
            solution = self.generate_solution(task)
            
            # 评估
            score = self.evaluate(solution)
            
            if score > best_score:
                best_score = score
                best_solution = solution
            
            # 反思
            reflection = self.reflect(solution, score)
            
            # 改进（下一轮使用反思）
            self.context.append(reflection)
        
        return best_solution
```

### NLP 生成时如何选择最优序列？

**解码策略对比：**

| 策略 | 描述 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| **Greedy** | 每步选概率最大 | 快速 | 局部最优 | 简单任务 |
| **Beam Search** | 保留 K 个候选 | 平衡质量/速度 | 可能重复 | 翻译/摘要 |
| **Sampling** | 按概率采样 | 多样性高 | 不稳定 | 创意写作 |
| **Top-K** | 从 K 个中采样 | 控制多样性 | 需调 K 值 | 通用 |
| **Top-P** | 从累积概率 P 中采样 | 自适应 | 需调 P 值 | 通用 |
| **Temperature** | 调整分布平滑度 | 灵活控制 | 需配合其他 | 通用 |

**推荐配置：**
