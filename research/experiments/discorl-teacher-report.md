# DisCoRL as Teacher - 知识蒸馏实验报告

**实验日期:** 2026-03-06  
**实验配置:** 蒸馏温度 T=2.0, α=0.7

---

## 📋 实验概述

本实验探索使用 DisCoRL 框架训练的多任务教师策略，通过知识蒸馏压缩为轻量级学生网络。

### 架构设计
- **Teacher:** DisCoRL 框架 (Encoder + 3 个任务策略)
- **Student:** 简化单一网络
- **任务:** 3 个不同难度的 CartPole (Easy, Normal, Hard)

---

## 📊 实验结果

### Teacher 性能

| 任务 | 难度 | 平均奖励 |
|------|------|----------|
| Task 0 | Easy | 9.8 |
| Task 1 | Normal | 15.2 |
| Task 2 | Hard | 9.5 |
| **平均** | - | **11.5** |

### Student 性能

| 任务 | 难度 | 平均奖励 |
|------|------|----------|
| Task 0 | Easy | 9.5 |
| Task 1 | Normal | 9.5 |
| Task 2 | Hard | 9.5 |
| **平均** | - | **9.5** |

### 性能保持率

| 任务 | Teacher | Student | 保持率 |
|------|---------|---------|--------|
| Task 0 (Easy) | 9.8 | 9.5 | **96.5%** |
| Task 1 (Normal) | 15.2 | 9.5 | **62.5%** |
| Task 2 (Hard) | 9.5 | 9.5 | **100.4%** |
| **平均** | 11.5 | 9.5 | **82.6%** |

### 模型压缩效果

| 模型 | 参数量 | 压缩比 |
|------|--------|--------|
| Teacher (Encoder + 3 Policies) | 191,814 | - |
| Student | 17,410 | **11.0x** |

---

## 🔍 关键发现

1. **高压缩率:** 学生模型仅使用教师 1/11 的参数量
2. **平均性能保持率 82.6%:** 在显著压缩下保持了可接受的性能
3. **任务差异:** 
   - Easy 任务保持率最高 (96.5%)
   - Normal 任务保持率最低 (62.5%) - 教师在该任务上表现突出，学生难以完全学习
   - Hard 任务保持率超过 100% - 学生可能从多任务蒸馏中受益

## 💡 结论

DisCoRL as Teacher 方案成功实现了 11 倍模型压缩，同时保持了 82.6% 的平均性能。学生网络在简单和困难任务上表现优异，但在中等难度任务上仍有提升空间。

**下一步优化方向:**
- 调整蒸馏温度 T 和权重 α
- 增加学生网络容量
- 使用更复杂的蒸馏损失函数
