## 🔒 VPN 隔离方案探讨 (14:05)
**问题:** 如何通过 subagent 实现 VPN 连接而不影响主 agent 网络配置

**结论:** Subagent 本身无法提供网络隔离 (共享同一网络命名空间)

**可行方案:**
1. **Docker 容器** - 独立 network namespace，容器内运行 VPN
2. **Linux network namespaces** - 创建隔离网络环境
3. **HTTP/SOCKS 代理** - 仅特定流量走代理，不影响全局
4. **SSH 动态转发** - `ssh -D 1080` 创建 SOCKS 代理

**推荐:** Docker 方案最安全隔离
