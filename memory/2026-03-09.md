## Reward Machine 论文调研

**时间**: 2026-03-09T17:59:00+08:00

### 核心论文

**"Using Reward Machines for High-Level Task Specification and Decomposition in Reinforcement Learning"**
- **作者**: Rodrigo Toro Icarte, Toryn Klassen, Richard Valenzano, Sheila McIlraith
- **会议**: ICML 2018 (Proceedings of the 35th International Conference on Machine Learning)
- **链接**: https://proceedings.mlr.press/v80/icarte18a.html
- **PDF**: http://proceedings.mlr.press/v80/icarte18a/icarte18a.pdf

### 核心贡献

1. **Reward Machines 定义**: 一种有限状态机，用于指定奖励函数，同时向学习者暴露奖励函数结构并支持分解

2. **Q-Learning for Reward Machines (QRM)**:
   - 适当分解 Reward Machine
   - 使用 off-policy Q-learning 同时学习不同组件的子策略
   - 在表格情况下保证收敛到最优策略
   - 与分层 RL 方法相比（可能收敛到次优策略）有理论优势

3. **实验验证**:
   - 两个离散域实验
   - 连续状态空间域（使用神经网络函数逼近）
   - 比分层方法更快找到更好策略

### 与 DRAMA 实现的对比

| 特性 | 原论文 (QRM) | DRAMA 实现 (Enhanced RM) |
|------|-------------|-------------------------|
| 状态机类型 | 有限状态机 (FSM) | 有限状态机 (FSM) |
| 状态定义 | 任务完成标签序列 | 任务完成比例 (0%, 25%, 50%, 75%, 100%) |
| 奖励分解 | 按 RM 状态分解 | 任务完成/协作/进度/效率 |
| 算法集成 | Q-Learning (QRM) | MAPPO (多 Agent PPO) |
| 协作机制 | 无 | 多 Agent 同时完成任务奖励 |
| 适用场景 | 单 Agent 任务 | 多 Agent 协作任务 |

### 设计参考

DRAMA 的 Enhanced RM 设计参考了原论文的核心思想:
1. **状态暴露**: 将任务进度显式编码为 RM 状态
2. **奖励分解**: 将复杂奖励分解为多个可解释组件
3. **课程学习**: 通过状态转移实现隐式课程学习

**创新点**:
- 扩展到多 Agent 场景 (MAPPO)
- 添加协作奖励机制
- 连续进度跟踪 (而非离散标签)
